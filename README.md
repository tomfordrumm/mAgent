# mAgent

Простой CLI-агент для работы с локальной кодовой базой через OpenAI Responses API. Проект запускается из терминала, принимает пользовательские запросы, умеет читать и изменять файлы внутри текущего проекта, просматривать структуру директорий и выполнять bash-команды в рамках рабочей папки.

Репозиторий: [git@github.com:tomfordrumm/mAgent.git](git@github.com:tomfordrumm/mAgent.git)

## Что это за проект

mAgent — это минималистичный JavaScript-агент для разработчиков, который работает прямо в локальном репозитории. Он поднимается как CLI-приложение, отправляет запросы в OpenAI API, получает ответы модели и при необходимости вызывает локальные инструменты для работы с проектом.

Агент подходит для простых задач по исследованию и редактированию кода:
- просмотр структуры проекта;
- чтение файлов;
- запись файлов;
- запуск bash-команд;
- ведение диалога в интерактивном режиме с сохранением контекста между сообщениями.

## Стек проекта

Проект собран на следующем стеке:

- **Node.js** — среда выполнения;
- **JavaScript (ES Modules)** — основной язык проекта;
- **OpenAI Responses API** — для генерации ответов и вызова инструментов;
- **dotenv** — для загрузки переменных окружения из `.env`;
- **readline/promises** — для реализации интерактивного CLI;
- **child_process** — для выполнения локальных bash-команд;
- **fs/promises** и **path** — для безопасной работы с файлами и путями внутри проекта.

## Возможности

Сейчас агент поддерживает следующие возможности:

### 1. Интерактивный CLI-режим
После запуска приложение открывает терминальный интерфейс, в котором можно отправлять запросы агенту в формате чата.

### 2. Работа с локальными файлами
Агент умеет:
- читать текстовые файлы внутри проекта;
- записывать и перезаписывать файлы;
- создавать директории при записи, если они отсутствуют.

### 3. Просмотр структуры проекта
Агент может выводить список файлов и папок относительно корня проекта.

### 4. Выполнение bash-команд
Можно запускать shell-команды внутри корневой директории проекта. Это полезно для диагностики, просмотра структуры файлов, запуска утилит и получения служебной информации.

### 5. Защита от выхода за пределы проекта
Все пути резолвятся относительно текущего project root, поэтому агент ограничивает работу файловыми операциями только внутри директории проекта.

### 6. Поддержка конфигурации через переменные окружения
Можно указать:
- `OPENAI_API_KEY` — ключ доступа к OpenAI API;
- `OPENAI_MODEL` — модель, используемую по умолчанию (если не указана, используется `gpt-5.2`).

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone git@github.com:tomfordrumm/mAgent.git
cd mAgent
```

### 2. Установка зависимостей

```bash
npm install
```

### 3. Настройка окружения
Создайте файл `.env` на основе примера:

```bash
cp .env.example .env
```

Заполните переменные:

```env
OPENAI_API_KEY=your_openai_api_key_here
OPENAI_MODEL=gpt-5.2
```

### 4. Запуск проекта

```bash
npm start
```

Или через бинарную команду:

```bash
npx my-agent
```

## Как использовать

После запуска в терминале появится приглашение ввода. Дальше можно писать задачи в свободной форме, например:

- `Покажи структуру проекта`
- `Прочитай package.json`
- `Создай README для проекта`
- `Запусти команду ls -la`

Для завершения работы введите:

```text
exit
```

## Структура проекта

- `agent.js` — основная логика CLI-агента;
- `package.json` — метаданные проекта, зависимости и скрипты;
- `.env.example` — пример файла окружения.

## Особенности реализации

- агент использует системный промпт с указанием текущего корня проекта;
- инструменты описаны как function tools и передаются в OpenAI API;
- ответы модели могут включать вызовы локальных инструментов;
- вывод bash-команд ограничивается по длине, чтобы не перегружать терминал;
- история диалога поддерживается через `previous_response_id`.

## Требования

- Node.js 18+;
- npm;
- доступ к OpenAI API;
- настроенный `OPENAI_API_KEY`.
